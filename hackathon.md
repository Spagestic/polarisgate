# Generative UI Global Hackathon

## What the hackathon is

- A global, in-person, six-hour hackathon hosted across AI Tinkerers cities.
- Focused on building agentic interfaces: AI apps where the agent generates interactive UI at runtime.
- The goal is to ship a working prototype by \(6{:}00\) PM local time.
- It includes judging, prizes, and a global winner pool.

## Core goal

Participants should build something that goes beyond chat and would be impossible as a plain chatbot.

Key objectives:

- Build and submit a working generative UI prototype.
- Learn at least one protocol hands-on: A2UI, AG-UI, or MCP Apps.
- Explore new UI patterns with other engineers.
- Submit to the global competition.

Main principle:

- Shipping matters more than polish.

## What counts

This event is for:

- Engineers building real working software.
- Hands-on experimentation with sponsor protocols and tools.
- Fast prototyping in a focused environment.

This event is not for:

- Pitching ideas.
- Panel discussions or recruiting.
- Repackaging a chatbot with a prettier interface.

## Theme and tracks

Theme: Agentic Interfaces

Build an AI app where the agent renders the UI dynamically at runtime.

Suggested tracks:

- Kill the Dashboard: agents generate the exact chart, form, or control surface needed on demand.
- The Copilot That Ships: copilots render UI for users to confirm, edit, and execute actions inline.
- Agent App Store: MCP-powered apps where agents discover and combine tools into generated UI experiences.
- No Designer, No Problem: the entire frontend is generated at runtime by the agent.

Guiding test:

- If the idea would work just as well in chat, it likely does not fit this hackathon.

Teams:

Support:

- Starter kits from CopilotKit and A2UI are allowed

## Submission requirements

Projects must be submitted through the global portal by \(6{:}00\) PM local time.

A valid submission needs:

- Project name
- One-sentence pitch
- Short description explaining what was built and why it is generative UI
- Public GitHub repo
- \(2\)–\(3\)-minute demo video
- Protocols used
- Team member names and roles

## Rules

- In-person attendance is required in a participating city.
- Teams can have up to 4 people.
- Participants keep ownership of IP created during the event.
- Pre-existing code is allowed, but teams must clearly state what was built during the six hours.
- Open-source tools, public APIs, and sponsor tech are allowed.
- Recruiting, product selling, and sponsor floor pitches are not allowed.
- Respect the focused nature of the event.

## Conduct and media

- The event emphasizes a respectful, inclusive, builder-first environment.
- Harassment, discrimination, or disruptive behavior can lead to removal and disqualification.
- Issues should be reported to organizers or emailed to [conduct@aitinkerers.org](mailto:conduct@aitinkerers.org).
- Photos and short videos may be taken for recap and promotion.
- Attending implies consent to appear in event media; removals can be requested via organizers or [media@aitinkerers.org](mailto:media@aitinkerers.org).

## Bottom line

This hackathon is about proving that AI can generate rich, task-specific, interactive interfaces at runtime — not just prettier chat apps. It’s a fast, global build sprint happening on May \(9\), with submission due by \(6{:}00\) PM local time.
